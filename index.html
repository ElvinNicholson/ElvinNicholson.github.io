Games I've worked on
TicTako
C#
Unity
WebGL
A Tic-Tac-Toe fan game of the VTuber group Hololive EN, also the first game I have ever released.

TicTako is a classic Tic-Tac-Toe game themed around the popular VTuber group Hololive EN. The game features a fully hand-drawn art made by me, with a lot of animated elements and can be played by two players with local multiplayer or solo against a bot. The bot uses a minimax algorithm to determine its best move, which is why its impossible to win against it. In hindsight, this was a horrible game design, however at that time, I had little to no knowledge about game programming or the Unity engine, so I made it so that the player only has to get a draw against the bot to win.

Overall, this was a very eye-opening experience for me into the world of game development and taught me a lot of valuable lesson on how to NOT program a game. The game took me around a month to create over the summer, and was released back on 4th August, 2021. As of the 21st March 2023, the TicTako has over 3,000 browser plays on itch io.

Unity Race Creator
C#
Unity
A designer facing tool that allows for the creation of a racing system within Unity without having to code anything.

The Unity Race Creator is a designer facing tool created within Unity which allows designers with no programming knowledge to create and modify a racetrack on a pre-existing game world. It has a custom window that can be used to create a new race and a custom inspector that can modify values such as the number and visuals of checkpoints, starting and ending gates, number of laps, timers, and more.

The system also comes with a simple UI that displays crucial information such as the timer, current lap, a waypoint to the next checkpoint and a minimap.

RayLib Race Creator
C++
RayLib
A system that allows for the creation of a racing system within RayLib through the JSON Level Editor.

The RayLib Race Creator is a counterpart of the Unity Race Creator built on RayLib. Raylib is a low-level game engine with no interface, visual helpers, or editors like Unity. Everything in Raylib is done through C++ scripting. The biggest challenge of this project was creating an easy-to-use environment for level designers to create and modify a race without having to interact with the source code of the game. To tackle this problem, the JSON Level Editor was made using Windows Presentation Foundation (WPF).

Race tracks in RayLib are loaded in on runtime using a JSON file that can be easily modified using the JSON Level Editor. The race system has the same features as the Unity Race Creator, with a few added features such as racing bots and a scoreboard. The bots have multiple difficulty settings and can pathfind to the next checkpoint smoothly.

JSON Level Editor
C#
XAML
WPF
An application made using WPF to easily create and modify a race for the RayLib Race Creator.

The JSON Level Editor is a tool made using Windows Presentation Form (WPF) to modify a JSON file used to load a race in the RayLib Race Creator. It allows level designers to create a new racetrack and modify them through a visual interface rather than staring at a wall of text in a JSON file. The editor features a scene view that shows all the game objects related to the race similar to Unity, with a side bar can modify these game objects such as changing their position, texture and model.

Afterlife in Unity
C#
Unity
A Unity rapid alpha build of the Afterlife game originally made by LucasArts, published on 1996.

Afterlife in Unity is a recreation of the Afterlife game made by LucasArts made in 2 weeks with a team of 6 people as a university project. The aim of the project is to create a rapid alpha build of the game in Unity as a preparation for porting it to DirectX 11. Essentially, Afterlife is an isometric city builder, but you are managing both heaven and hell which have different requirements to earn the most amount of money.

In this project, I was responsible for creating the entire building system and the functionalities for all the buildings in the game. The building system consists of the tilemaps for both planes, how the player interacts with the map, the visuals of the building, the zone system, the roads, etc.

Mobile
A dialogue based mobile puzzle game that touches on the complexities of therapy in a lighter tone.

Get Therapy is a mobile puzzle game made in Unity which deals with the complexities of therapy in a video game format. The game was made by a group of 5 people, and I was in charge of creating the entire title and level selection of the game, the sprites and animations of both characters present in the game, the system that changes the colour of the background depending on the patients mood and building the game to mobile.

Boss Beater
Python
PyASGE
A 2D turn-based PVE game with multiple maps, enemies and bosses, made in Python using PyASGE.

Boss Beater was made in a group of 3 people as a university project. The objective of the game is to defeat the enemies in the desert, beach, and graveyard map without dying. The game features a unique fully animated boss with a behaviour tree for each map and some simpler enemies. The player can only perform two actions per turn, such as a ranged attack, a melee attack and moving to a different position.

I was responsible for all the UI elements of the game, such as the health and stamina bar, the enemy health bar, the turn order shown on the right side of the screen, the title and game over screens. I was also in charge of creating all the bosses (Minotaur, Pirate and Leshen) in the game including animating them and giving them unique behaviours.
